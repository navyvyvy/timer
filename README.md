# Step Timer

로스팅, 만두, 라면처럼 단계가 있는 작업을 놓치지 않도록 만든 정적 웹 타이머입니다. 각 타이머는 독립 프로필로 관리되고, 카운트다운, 단계 알람, 연장, 같은 리스트 재시작을 한 화면에서 다룹니다.

## 빠른 시작

브라우저에서 바로 열 수 있습니다.

```text
index.html
```

홈 화면 추가나 오프라인 캐시까지 확인하려면 간단한 로컬 서버로 열면 됩니다.

```bash
python -m http.server 8000
```

```text
http://localhost:8000/index.html
```

## 핵심 흐름

### 타이머 프로필

- 상단에서 `로스팅`, `만두`, `라면` 전환
- 진행 중에는 모드 전환과 광고 영역을 숨겨 클릭 실수를 줄임
- 각 프로필은 별도 저장 키로 예열 시간, 목표 온도, 단계 구성을 보존
- 기존 로스팅 저장값은 새 로스팅 프로필에서도 읽을 수 있음

### 라면 제품 프리셋

- 라면 프로필에서 30개가 넘는 유명 봉지라면 프리셋 제공
- 기본 정렬은 인기순이며, `회사별`과 `끓이는 라면 / 비비는 라면` 필터를 유지함
- 즐겨찾기 별표와 즐겨찾기만 보기 필터 제공
- 제품 선택 시 1봉 기준, 총 조리 시간, 조리법, 설명, 출처 링크 표시
- 끓이는 라면과 비비는 라면은 각 제품의 첫 조리 단계를 기준으로 타이머를 맞춤
- 사용자가 시간을 조절한 뒤에도 `기본값 적용`으로 선택 제품 기준을 다시 불러올 수 있음
- 선택한 라면 제품, 필터 상태, 즐겨찾기, 조정한 마일스톤은 브라우저에 저장

### 만두 제품 프리셋

- 만두 프로필에서 국내 냉동만두와 동원 딤섬 프리셋 제공
- `냉동 / 해동` 기준을 전환하고, 회사별 필터와 인기순 / 가나다순 정렬을 유지함
- 즐겨찾기 별표와 즐겨찾기만 보기 필터 제공
- 제품 선택 시 찜기 기준 설명, 총 시간, 출처 링크, 기본 마일스톤 표시
- 동원 딤섬 샤오롱바오, 새우하가우, 부추창펀, 부채교를 포함함
- 선택한 만두 제품, 필터 상태, 즐겨찾기, 냉동 / 해동 기준은 브라우저에 저장

### 예열 타이머

- 로스팅 프로필에서만 표시
- 기본 10분 카운트다운
- 목표 온도 입력
- 실행 중 시간 보정: `-1분`, `-30초`, `-10초`, `+10초`, `+30초`, `+1분`
- 완료 시 화면 점멸, 소리, 진동 알람

### 집중 화면

- 시작 전에는 총 시간을 크게 표시
- 시작 후에는 중앙 숫자를 `다음 행동까지` 남은 시간으로 전환
- 총 시간은 별도 참조 줄로 계속 표시
- 진행 중에는 현재 마일스톤만 강조하고 다른 단계는 숨김
- 현재 마일스톤의 기준 시간과 보정 버튼은 계속 표시
- 완료 후에는 이번 타이머의 실제 시간을 중앙에 유지

### 마일스톤 관리

- 로스팅 기본 단계: `뒤집기`, `상태 체크`, `로스팅 완료`
- 만두 기본 단계: 선택한 제품 프리셋의 찜기 기준 냉동 / 해동 조리
- 라면 기본 단계: 선택한 제품 프리셋의 조리 기준
- 각 단계는 독립 구간 시간으로 계산
- 마일스톤 이름 변경
- 마지막 완료 단계 앞에 새 타이머 추가
- 도달하지 않은 중간 마일스톤 삭제
- 진행 중에도 현재 단계 시간 보정 가능
- 프로필별 예열 시간, 목표 온도, 마일스톤 구성을 브라우저에 자동 저장

자동 저장 대상은 레시피에 해당하는 설정만입니다. 진행 중인 시간, 알람 큐, 완료 여부 같은 일회성 상태는 새로고침 후 복원하지 않습니다.

### 알람 처리

- 화면 전체 점멸
- 반복 비프음
- 진동 지원 환경에서 하드웨어 진동
- 알람 확인 전까지 전면 모달 유지
- 여러 알람이 겹치면 큐에 쌓아 순서대로 표시
- 모달 포커스 고정과 초기화 취소 `Esc` 처리

### 완료와 연장

- 완료 시 `3분`, `4분`, `5분`, `6분` 추가 선택
- 추가 시간 선택 시 총 시간에 반영
- 참조 줄에 `+4분 추가됨`처럼 연장 내역 표시
- 완료 후에는 현재 리스트를 유지한 채 다시 시작 가능
- `초기화`는 기본 리스트와 기본 시간으로 되돌림

### 광고와 사이트 구조

- 앱 아래에 광고 슬롯 자리 준비
- 타이머 진행 중에는 광고 슬롯을 숨겨 조작부 근처 오클릭을 방지
- footer에 `개인정보처리방침`, `문의` 링크 제공
- 실제 AdSense 적용 시 게시자 ID와 광고 슬롯 ID를 별도로 설정해야 함

### 앱 설치와 오프라인 사용

- `manifest.webmanifest`와 `icon.svg` 포함
- 지원 브라우저에서는 홈 화면에 추가 가능
- `service-worker.js`로 기본 파일 캐시
- `file://`로 열 때는 서비스워커 등록을 건너뜀

서비스워커는 `http://localhost` 같은 서버 환경에서만 등록됩니다. 파일을 직접 열어도 타이머 기능은 그대로 동작합니다.

## 파일 구성

- `index.html`: 앱 UI, 스타일, 타이머 로직
- `privacy.html`: 개인정보처리방침과 광고 쿠키 고지
- `contact.html`: 문의 안내
- `manifest.webmanifest`: 설치형 앱 메타데이터
- `service-worker.js`: 기본 파일 캐시
- `icon.svg`: 앱 아이콘

## UI 방향

- 어두운 조리 도구형 팔레트 유지
- 로스팅, 만두, 라면은 선택 상태와 주요 행동 색으로 구분
- 완료 상태는 초록, 위험/긴급 상태는 빨강으로 분리
- 버튼과 상태 라벨은 굵은 글씨로 가독성 강화
- 모바일 터치 타깃을 고려한 버튼 높이 적용

## 기술 메모

- 빌드 도구 없는 단일 `index.html`
- 순수 HTML, CSS, JavaScript
- `Date.now()` 기반 시간 계산
- `requestAnimationFrame` 단일 루프에서 예열과 선택된 타이머 관리
- `Web Audio API`로 알람음 생성
- `Screen Wake Lock API` 지원 환경에서 화면 유지
- `prefers-reduced-motion` 대응
- 진행바는 `transform: scaleX()`로 갱신

## 검증 메모

- 모바일 기준 폭 `390px`에서 가로 스크롤 없이 표시
- 저장 후 새로고침 시 선택 프로필과 마일스톤 구성 복원
- 완료 후 총 시간 표시와 같은 리스트 재시작 확인
- 라면 제품 선택 시 총 시간, 설명, 출처 링크, 기본 마일스톤 전환 확인
- 로컬 서버 환경에서 서비스워커 활성화 확인

## 개발용 기능

하단 `DEV` 버튼으로 알람 미리보기 패널을 열 수 있습니다. 타이머를 실제로 시작하지 않아도 현재 프로필의 마일스톤 알람과 완료 연장 팝업을 테스트할 수 있습니다.
